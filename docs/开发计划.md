# Signal Track 开发计划

## 目标

把非结构化投资信息转成可跟踪项目，并自动维护价格曲线、跟踪逻辑、每日检查结论和网页看板。

第一阶段重点不是做复杂交易系统，而是把下面四件事打牢：

1. 标的识别：从中文/英文/代码/简称里识别唯一证券或期货合约。
2. 行情获取：支持 A 股、港股、中国期货、美股、美股期货的历史日线能力。
3. 逻辑生成：优先使用原始信息逻辑；逻辑不足时自动用 3C-5M-3D-3T 框架补全。
4. 看板发布：生成静态 HTML，通过演示管理发布 API 覆盖更新。

## 行情数据源判断

### 推荐优先级

| 市场 | MVP 数据源 | 正式数据源 | 说明 |
| --- | --- | --- | --- |
| A 股 | Tushare `daily` | Tushare | 官方文档显示 A 股日线接口支持按代码、日期区间取开高低收、成交量、成交额。 |
| 港股 | Tushare `hk_daily` | Tushare / Futu OpenAPI | Tushare 港股日线需要单独权限；Futu OpenAPI 可作为备选。 |
| 中国期货 | Tushare `fut_daily` + `fut_mapping` | Tushare | 需要同时处理具体合约与主力/连续合约映射。 |
| 美股 | Tushare `us_daily` / Futu OpenAPI | Polygon/Massive 或 Tushare | 美股暂时非主战场，可先做适配层，不强依赖付费源。 |
| 美股期货 | yfinance 仅作临时兜底 | CME DataMine / 专业期货数据源 | CME DataMine 是更正式的历史期货数据渠道；IBKR 不适合作为长期历史数据库。 |

### 已确认资料

- Tushare A 股日线 `daily`：交易日 15:00-16:00 入库，支持代码、交易日、开始/结束日期，返回 OHLC、涨跌幅、成交量、成交额。
- Tushare 港股日线 `hk_daily`：每日 18 点左右更新，返回 OHLC、成交量、成交额，但需要单独权限。
- Tushare 期货日线 `fut_daily`：支持具体合约代码、交易所、日期区间，返回收盘价、结算价、成交量、持仓量等。
- Tushare 期货主力与连续合约 `fut_mapping`：用于把连续合约映射到实际月合约。
- Tushare 美股日线 `us_daily`：返回美股 OHLC、成交量、VWAP、PE、PB、市值等字段。
- Futu OpenAPI `request_history_kline`：可获取历史 K 线，支持分页；港股/A 股默认北京时间，美股默认美东时间。
- CME DataMine：提供购买后的历史期货数据 REST API；适合作为美股期货的正式来源。
- IBKR 历史行情有 pacing 限制，且过期期货超过两年不可用，不建议作为主历史数据源。

## 架构设计

### 1. 标的主数据层

建立 `instruments` 表，统一存放股票、ETF、指数、期货、连续合约：

```text
id
symbol
provider_symbol
name
aliases
market: CN_A | HK | CN_FUT | US | US_FUT
asset_type: stock | etf | index | future | continuous_future
exchange
currency
timezone
status
metadata
```

标的识别流程：

1. LLM 从原文抽取候选名称、代码、市场、资产类型。
2. 先用本地主数据精确匹配代码。
3. 再用别名、中文名、英文名模糊匹配。
4. 若多候选，按信息源上下文、市场关键词、常见 ticker 优先级自动选择，并打 `resolution_confidence`。
5. 如果仍低置信度，也先创建跟踪项，状态标为 `needs_review`，但不阻塞流程。

### 2. 行情适配层

定义统一接口：

```text
get_daily_bars(instrument_id, start_date, end_date, adjustment)
refresh_instrument_master(market)
resolve_symbol(raw_symbol, market_hint)
```

统一输出：

```text
date
open
high
low
close
adj_close
volume
amount
settle
open_interest
provider
provider_symbol
```

期货特殊处理：

- 如果输入是具体合约，例如 `CU2601.SHF`，直接取 `fut_daily`。
- 如果输入是主力/连续，例如“铜主连”，先用 `fut_mapping` 找每天对应的实际合约，再拼接连续曲线。
- 组合曲线使用权重加权收益率，不直接加权价格。

### 3. 信号与跟踪项目

核心表：

```text
sources
raw_inputs
tracking_projects
project_legs
logic_blocks
daily_checks
price_bars
publish_events
```

跟踪项目状态：

```text
active
exit_signal
closed
watch_after_close
archived
needs_review
```

逻辑处理规则：

- 原文逻辑够用：直接保存为 `source_logic`。
- 原文逻辑简单或不足：自动生成 `system_logic`，使用 3C-5M-3D-3T 补全。
- 原文几乎没有逻辑：仍创建项目，系统补全逻辑，并把置信度标低。
- 默认少问用户；只有组合权重严重缺失且无法合理默认时，先按等权创建，并标记 `weight_needs_review`。

### 4. 每日检查

调度建议：

- A 股、港股、中国期货：北京时间 19:00 后检查。
- 美股、美股期货：北京时间次日早上检查，或单独任务。
- 跨市场组合：按最晚市场可用数据时间检查。

每日检查内容：

1. 拉取缺失行情。
2. 更新项目收益曲线。
3. 对照原始逻辑与系统补充逻辑。
4. 输出 `hold / watch / exit_signal / needs_review`。
5. 生成并发布网页。

### 5. 网页看板

页面结构：

- 总览：活跃项目、平仓信号、今日更新、按信息源统计。
- 项目列表：信息源、标的、方向、入场日期、收益、状态、逻辑置信度。
- 项目详情：原始信号、系统补充逻辑、价格曲线、每日检查日志。
- 组合详情：组合收益曲线 + 每个 leg 的单独曲线。

视觉方向：

- 遵循根目录 `DESIGN.md`。
- Card based design with layered elements。
- Futuristic minimalism with glassmorphism。
- 数据密集、克制、可扫读，避免营销页式 hero 和纯装饰视觉。

发布方式：

- 生成单页 HTML。
- 调用 `POST /api/demos/publish`。
- 固定 `title`，同名覆盖，保留 URL。

## 开发阶段

### Phase 0：项目骨架

- 选择技术栈：Python + FastAPI + SQLite/Postgres + APScheduler + Jinja/静态 HTML。
- 建立配置管理：`.env`、provider token、发布 API。
- 建立数据库迁移。

验收标准：本地服务可启动，数据库可初始化，配置不进 git。

### Phase 1：行情与标的能力

- 实现 Tushare provider。（已实现 daily bars + instrument master）
- 建立 A 股、港股、中国期货、美股的 symbol resolver。（已实现 seed + refresh 后 resolver）
- 实现 daily bars 标准化。（已实现）
- 实现期货主力/连续合约映射。（已实现 Tushare `fut_mapping` 路径）
- 加入缓存和去重，避免重复请求。

验收标准：给定 5 个样例标的，能返回最近 2 年日线并写入本地库。

### Phase 2：信息抽取与项目创建

- 定义 Structured Outputs schema。（已实现，见 `src/signal_track/extraction.py`）
- 实现 raw input -> candidate signals -> tracking projects。（已实现 heuristic + OpenAI 可选路径）
- 实现组合识别、等权默认、权重标记。
- 实现逻辑充足度评分和自动补全。

验收标准：粘贴一段包含多个标的的信息，能自动创建多个项目或一个组合项目。

### Phase 3：每日检查

- 实现定时任务。
- 实现价格更新、收益计算、触发条件检查。
- 实现 daily check 记录。

验收标准：手动触发一次检查后，所有活跃项目都有检查结论。

### Phase 4：网页发布

- 实现 HTML 看板。
- 实现项目详情、组合曲线。
- 实现发布 API 客户端。

验收标准：新建项目或每日检查后，远端页面被覆盖更新。

### Phase 5：深度研究增强

- 接入联网研究流程。
- 对低逻辑项目自动生成 3C-5M-3D-3T 跟踪指标。
- 保存数据来源、交叉验证记录和未验证标记。

验收标准：逻辑不足的项目也能自动生成可执行跟踪指标，而不是只留下空泛观点。

## 下一步执行建议

先做 Phase 0 + Phase 1。原因是行情和标的识别是整个系统的地基；一旦这里稳定，后面的信号抽取、网页、定时任务都可以自然接上。

建议第一批样例标的：

- A 股：`300750.SZ` 宁德时代、`600519.SH` 贵州茅台
- 港股：`00700.HK` 腾讯控股、`09988.HK` 阿里巴巴-W
- 中国期货：`CU.SHF` 铜主连、`IF.CFX` 沪深300股指主连
- 美股：`AAPL`、`NVDA`
- 美股期货：`ES`、`NQ`

美股期货在 MVP 中可以先完成接口抽象和 symbol 设计，正式数据源后接。不要把 yfinance 这类临时数据当成长期可信数据源。
