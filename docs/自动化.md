# Codex App 自动化

## 设计原则

Signal Track 的定时检查由 Windows Codex App Automations 承担。自动化唤醒本仓库工作区后，由 Codex 在当前项目里调用本地 CLI 完成检查、渲染和发布。

这不是后台服务。自动化只是定时让 Codex 执行一次本地工作流。

## 推荐自动化

### 每日主检查

- 时间：19:00 Asia/Shanghai。
- 用途：检查 A 股、港股、中国期货和所有一般跟踪项目。

建议提示词：

```text
在本工作区执行 Signal Track 每日检查并发布：
python -m signal_track.cli daily-run --provider auto --archive-reports --publish

如果 provider auto 不可用，降级执行：
python -m signal_track.cli daily-run --provider none --archive-reports --publish

请总结检查项目数、触发的平仓信号、发布结果和需要我手动处理的事项。
```

### 美股补跑

- 时间：07:00 Asia/Shanghai。
- 用途：可选，用于美股和美股期货收盘后的补充检查。

建议提示词：

```text
在本工作区执行 Signal Track 美股补跑检查并发布：
python -m signal_track.cli daily-run --provider auto --archive-reports --publish

重点说明美股、美股期货相关项目的新增检查记录、退出信号和发布结果。
```

## 手动触发

临时检查：

```powershell
python -m signal_track.cli daily-run --provider auto --archive-reports --publish
```

只渲染不发布：

```powershell
python -m signal_track.cli daily-run --provider auto --no-publish
```

行情不可用时保守检查并发布：

```powershell
python -m signal_track.cli daily-run --provider none --archive-reports --publish
```
