# 演示管理接口

`cmd/demo-server` 提供两类演示管理能力：

- 浏览器里的后台管理：登录后通过 HttpOnly cookie 上传、改名、禁用、删除演示。
- 外部项目发布 API：通过 API Key 发布一个网页，同名标题会直接覆盖原演示。

## 后台管理路由

- `GET /api/demo/session`：检查当前 HttpOnly cookie 是否仍有效。
- `POST /api/demo/session`：提交 `{ "password": "..." }`，成功后设置 HttpOnly cookie。
- `GET /api/demos`：列出演示。已登录时包含禁用项，未登录时只返回启用项。
- `POST /api/demos`：multipart 上传，字段为 `file` 和可选 `title`。支持 `.html`、`.md`、`.zip`。
- `PATCH /api/demos/{slug}`：提交 `{ "disabled": true | false }`、`{ "title": "新名称" }` 或 `{ "feature": "功能说明" }`。
- `DELETE /api/demos/{slug}`：删除演示记录和对应文件。
- `GET /demo/{slug}/...`：由后端服务演示文件；禁用项返回 404。

## 外部发布 API

`POST /api/demos/publish`

当前可用配置：

```text
GO_SITES_DEMO_PUBLISH_URL=https://liangz77.cn/api/demos/publish
GO_SITES_DEMO_API_KEY=<your-demo-api-key>
```

认证方式二选一：

- `Authorization: Bearer <your-demo-api-key>`
- `X-Demo-API-Key: <your-demo-api-key>`

请求体为 JSON：

```json
{
  "title": "项目演示",
  "html": "<!doctype html><html><body>hello</body></html>",
  "feature": "来自外部项目的自动发布",
  "disabled": false
}
```

字段说明：

- `title`：必填。会作为演示名称；如果已有同名标题，直接覆盖原文件并保留原 URL。
- `html`：HTML 页面内容。适合外部项目直接发布构建后的单页 HTML。
- `markdown`：Markdown 内容。传它时服务端会渲染成演示页。
- `content` + `kind: "markdown"`：等价于 `markdown`。
- `content`：未指定 `kind` 时按 HTML 处理。
- `slug`：可选，仅首次创建时生效。同名覆盖时始终复用原 slug。
- `feature`：可选，显示在演示 tab 的功能说明列。
- `disabled`：可选。覆盖同名演示且不传时，会保留原禁用状态。

返回：

- 首次创建返回 `201 Created` 和演示条目。
- 同名覆盖返回 `200 OK` 和更新后的演示条目。
- API Key 未配置返回 `503 API_AUTH_NOT_CONFIGURED`。
- API Key 错误或缺失返回 `401 UNAUTHORIZED`。

curl 示例：

```bash
curl -X POST https://liangz77.cn/api/demos/publish \
  -H "Authorization: Bearer $GO_SITES_DEMO_API_KEY" \
  -H "Content-Type: application/json" \
  -d @payload.json
```

从项目构建产物发布单个 HTML 文件：

```bash
jq -n --rawfile html dist/index.html \
  '{ title: "项目演示", html: $html, feature: "自动构建发布" }' \
  | curl -X POST https://liangz77.cn/api/demos/publish \
      -H "Authorization: Bearer $GO_SITES_DEMO_API_KEY" \
      -H "Content-Type: application/json" \
      -d @-
```

也可以在别的项目里把上面的 key 放进环境变量，再用下面的通用脚本：

```bash
export GO_SITES_DEMO_PUBLISH_URL="https://liangz77.cn/api/demos/publish"
export GO_SITES_DEMO_API_KEY="<your-demo-api-key>"

jq -n --arg title "项目演示" --rawfile html dist/index.html \
  '{ title: $title, html: $html, feature: "自动构建发布" }' \
  | curl -X POST "$GO_SITES_DEMO_PUBLISH_URL" \
      -H "Authorization: Bearer $GO_SITES_DEMO_API_KEY" \
      -H "Content-Type: application/json" \
      -d @-
```

## 运行期配置

生产环境建议设置：

```text
DEMO_DATA_DIR=/srv/apps/go-sites-demo/shared
SITE_ROOT=/srv/sites/liangz77.cn/current
DEMO_ADMIN_PASSWORD=<server admin password>
DEMO_API_KEY=<server demo api key>
DEMO_SESSION_SECRET=<server session secret>
PUBLIC_ORIGIN=https://liangz77.cn
```

上传内容会保存到 `DEMO_DATA_DIR/demos/{slug}`，清单保存到 `DEMO_DATA_DIR/manifest.json`。
