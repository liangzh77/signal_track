# Signal Track 完成审计

本文档用于把用户需求拆成可验证项，避免把“已有进展”误判为“全部完成”。

## 当前结论

截至 2026-06-07，代码和文档已经切换为 Codex App-first 架构：

- Windows Codex App 是唯一的 AI 分析、调度和执行入口。
- 本仓库不提供 FastAPI、Web Inbox、后台 OpenAI 调用或常驻 scheduler。
- 本仓库提供确定性的本地 CLI：SQLite 状态、行情 provider、规则检查、Markdown 报告归档、静态 dashboard 渲染和外部发布 API 上传。
- 发布网站是单个静态 HTML，分析和调度都不在网页里执行。
- 数据采用 SQLite + Markdown 混合架构：SQLite 是运行状态事实源，Markdown 是人类可读的投研档案。
- 已在 Codex App 创建 19:00 Asia/Shanghai 的每日检查与发布自动化，automation id 为 `signal-track`。
- 当前环境已安装 `market,files` extras；`doctor` 可验证发布配置、数据库、skill 和各市场行情就绪度。

## 尚需外部配置或运营动作

这些不是代码缺口，也不阻止当前 Codex-first 工作流运行；它们只影响特定外部服务或生产级行情质量：

- 在本地 `.env` 写入真实 `GO_SITES_DEMO_API_KEY`，必要时写入 `TUSHARE_TOKEN`。
- 若要让中国期货自动刷新行情，需要在 `.env` 写入 `TUSHARE_TOKEN` 或接入 licensed provider；已有 CSV 时可先用 `import-bars` 导入。
- 若要生产级覆盖港股期货和美股期货，应接入 licensed provider；当前 yfinance 更适合作为临时 fallback。
- 真实信息源进入项目时，Codex 仍需按 skill 执行联网核验和 3C-5M-3D-3T 补充，这部分发生在 Codex 对话里，不在 repo 代码里。
